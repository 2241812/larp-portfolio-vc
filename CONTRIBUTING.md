# Contributing to 3D Portfolio

Thank you for your interest in contributing! This is a personal portfolio project, but contributions are welcome.

## Guidelines

### Reporting Bugs

Before creating bug reports, please check the issue list as you might find out that you don't need to create one:

- **Use clear, descriptive titles** for issues
- **Describe the exact steps which reproduce the problem** in as many details as possible
- **Provide specific examples** to demonstrate the steps
- **Describe the behavior you observed** and **what exactly is the problem**
- **Explain which behavior you expected** to see instead

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion:

- **Use a clear, descriptive title**
- **Provide a step-by-step description** of the suggested enhancement
- **Provide specific examples** to demonstrate the steps
- **Describe the current behavior** and **the expected behavior**

### Pull Requests

- Fill in the required template
- Follow TypeScript and styling conventions
- End all files with a newline
- Include appropriate tests
- Update documentation as needed

## Development Setup

1. Fork and clone the repository:
```bash
git clone https://github.com/yourusername/my-3d-portfolio.git
cd my-3d-portfolio
```

2. Install dependencies:
```bash
npm install
```

3. Create a feature branch:
```bash
git checkout -b feature/your-feature-name
```

4. Make your changes and test:
```bash
npm run dev
npm run lint
npm run build
```

5. Commit with clear messages:
```bash
git commit -m "feat: add feature description"
git commit -m "fix: fix description"
git commit -m "docs: update documentation"
```

6. Push to your fork and submit a pull request

## Code Style

- Use TypeScript for all code
- Follow existing code patterns
- Use descriptive variable and function names
- Add comments for complex logic
- Format code with Prettier (auto-run on commit)

## Commit Messages

Follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:

- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `style:` for code style changes
- `refactor:` for code refactoring
- `perf:` for performance improvements
- `test:` for test changes
- `chore:` for build/dependency changes

Example:
```
feat: add dark mode toggle to settings
fix: resolve contributing calendar fetch timeout
docs: update deployment instructions
```
